# リブランド計画: Uber特化 → 売れ筋横断PRハブ

## 目標
「Uber副業ブログ」を、ジャンル問わず"売れそうな商品"を横断でPRする
**ハイブリッド型（ランキング/比較 × 診断）アフィリエイトハブ**に作り替える。

## 原則
- 既存資産（Uber4記事＋診断＋時給マップ）は**消さない**。SEO equityを保持し、
  「副業・節約」カテゴリの1つとして残す。
- テーマ無しの雑多PRは避け、"売れ筋を診断・比較・ランキングで刈る"仕組みで量産する。
- ブランド名・カテゴリ等は `src/site.ts` に中央化（1箇所で変更可能に）。

## ブランド（暫定・site.tsで変更可）
- 名称: 売れ筋ナビ
- タグライン: "今どれが買い？"を診断とランキングで解決する買い物ガイド
- ドメインは hiroblog-lifehack.com のまま。

## Phase 0（土台リブランド）★今回
- `src/site.ts` 新設: SITE_NAME / TAGLINE / DESCRIPTION / ナビ・カテゴリ定義
- Header / Footer / BaseLayout(meta,JSON-LD) / index を site.ts 参照の汎用に
- ヒーローを「診断・ランキングで選ぶ」ハブに、Uber前提コピーを撤去
- `/shindan` 診断ハブ、`/ranking` ランキングハブの器を用意（404回避）
- 既存カテゴリ(uber/credit-card/tax)ルートは維持

## Phase 1（ランキング量産基盤）
- RankingList コンポーネント（順位バッジ＋比較）
- Amazon売れ筋/急上昇を素材に、ジャンル横断ランキング記事を数本（検証済ASIN）
- カテゴリ体系: ガジェット/家電・生活/節約・副業(Uber含む)/本 等

## Phase 2（診断ハブ拡張）
- 2本目以降の診断（ガジェット選び等）、/shindan 一覧充実

## Phase 3（SEO/量産）
- sitemap有効化、カテゴリページ整備、内部リンク、OG画像

## ASIN運用（確立済み）
- 実在検証: `curl .../dp/ASIN` の title を見て判定（死=「ページが見つかりません」）
- エバーグリーンはASIN直、改訂/多SKU品は `amazonSearch()`（tag自動付与）
